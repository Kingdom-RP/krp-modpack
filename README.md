# krp-modpack

Состав модпака **Kingdom RP** (Minecraft 1.21.1 + NeoForge 21.1.233) — сторонние
моды поверх основного мода [`krp-mod`](https://github.com/Kingdom-RP/krp-mod).

## Как это устроено

- **Сами jar'ы НЕ в git** — они заливаются в **[Releases](../../releases)** этого
  репозитория (тег вида `pack-vN`). В git только `modlist.toml`.
- **`modlist.toml`** — декларативный список: на каждый мод имя файла, URL ассета
  в Releases и (опц.) SHA-256.
- Сборщик дистрибутива лаунчера (`krp-launcher/builder`, флаг `--modlist`) читает
  этот файл, скачивает/сверяет каждый jar и вписывает его в `manifest.json` с
  внешним url. Лаунчер качает моды напрямую отсюда в `mods/` игрока.

## Обновление состава

1. Залить нужные jar'ы в новый Release (напр. `pack-v2`).
2. Прописать/обновить записи в `modlist.toml` (url на новый тег, при желании sha256).
3. Пересобрать дистрибутив лаунчера с `--modlist` → новый `manifest.json` на Pages.

## ⚠️ Лицензии

Многие моды распространяются как **All Rights Reserved** и запрещают ре-хостинг.
Размещение их jar в этих Releases — это ре-хостинг. Проверяй условия каждого мода;
спорные оставляй ссылкой на оригинальный CDN (Modrinth/CurseForge) прямо в `url`.
